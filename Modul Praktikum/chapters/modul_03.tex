\section{Tujuan dan Output Praktikum}

\subsection{Tujuan Praktikum}
Setelah menyelesaikan praktikum ini, mahasiswa diharapkan mampu:
\begin{enumerate}
	\item Memahami peran Sistem Operasi Linux secara umum, termasuk komponen inti seperti \textit{kernel} dan \textit{shell}.
	\item Menjelaskan struktur direktori dan bernavigasi menggunakan Antarmuka Baris Perintah (CLI).
	\item Menggunakan sintaks dasar sistem operasi Linux beserta \textit{flag} dan argumennya.
	\item Menggunakan perintah dasar seperti \texttt{ls}, \texttt{cd}, \texttt{mkdir}, \texttt{rm}, \texttt{cp}, \texttt{mv}, \texttt{touch}, dan \texttt{cat}.
	\item Memahami konsep dasar kepemilikan dan hak akses (\textit{permissions}) pada sistem operasi berbasis Unix.
	\item Menerapkan perubahan hak akses menggunakan \texttt{chmod}, \texttt{chown}, serta memahami konsep otorisasi \texttt{sudo} dan \textit{superuser}.
\end{enumerate}

\subsection{Output Praktikum}
Pada akhir praktikum ini, mahasiswa diharapkan menghasilkan:
\begin{itemize}
	\item Direktori dan file rekayasa yang berhasil dibuat, disalin, dipindahkan, dan dimanipulasi melalui terminal CLI.
	\item Log atau dokumentasi perubahan hak akses pada file menggunakan kombinasi izin \textit{read, write, execute}.
	\item Pemahaman komprehensif terkait perbedaan operasi pengguna biasa (\texttt{\$}) dibandingkan dengan \textit{superuser} (\texttt{\#}).
\end{itemize}

\section{Dasar Teori}

\subsection{Arsitektur Dasar: Kernel, Shell, dan Terminal}
Linux adalah sistem operasi berbasis Unix yang bersifat \textit{open-source}, artinya kode sumbernya tersedia untuk publik dan siapa saja dapat mengaksesnya, mengubahnya, atau menyebarluaskan ulang \cite{das2012unix}. 
Dalam arsitekturnya, Linux memiliki \textit{kernel} yang terdiri dari sejumlah modul dan komponen yang membentuk bagian inti sistem operasi \cite{Silberschatz2018}. Kernel bertanggung jawab untuk mengatur komunikasi antara perangkat keras dan perangkat lunak, serta mengatur penggunaan memori dan prosesor. 

Untuk berinteraksi dengan kernel, pengguna menggunakan antarmuka baris perintah atau Terminal \cite{shotts2019linux}. Perintah pada Linux dieksekusi melalui \textit{shell}, yaitu program yang menyediakan lingkungan kerja bagi pengguna untuk berinteraksi dengan sistem operasi melalui perintah-perintah yang dijalankan pada terminal. 

\subsection{Sistem File dan Sintaks Perintah}
File system pada Linux disusun dalam sebuah hirarki dengan root (\texttt{/}) sebagai induknya, dan setiap direktori dan file memiliki hak akses yang dapat dikonfigurasi oleh pengguna \cite{das2012unix}. 
Saat berinteraksi di Terminal, setiap perintah memiliki struktur sintaks yang terdiri dari Perintah Dasar, \textit{Flag}, dan Argumen:
\begin{itemize}
	\item \textbf{Argument}: Nilai atau data yang diperlukan oleh perintah untuk menjalankan tugasnya \cite{shotts2019linux}. Argument dapat berupa file, direktori, atau opsi lain.
	\item \textbf{Flag}: Opsi tambahan pada perintah yang memberikan perintah tambahan yang dapat mempengaruhi cara kerja perintah. Flag dinyatakan dengan menggunakan tanda "-" (misalnya \texttt{ls -l}).
\end{itemize}

\subsection{Hak Akses (Permissions), Kepemilikan, dan Superuser}
Setiap file dan direktori memiliki "gembok" hak akses yang dibagi menjadi tiga jenis izin utama: \textit{Read} (r), \textit{Write} (w), dan \textit{Execute} (x). Untuk mengubah gembok tersebut digunakan perintah \texttt{chmod}, dan untuk mengubah pemilik digunakan \texttt{chown}.

Beberapa perintah pada Linux memerlukan hak \textit{superuser} atau hak administratif. Pengguna harus masuk sebagai root atau menggunakan perintah \texttt{sudo} untuk memberikan hak superuser pada perintah \cite{shotts2019linux}. 

Perhatikan tanda \textit{prompt} pada terminal Anda:
\begin{itemize}
	\item Tanda pagar \texttt{\#} menandakan Anda sedang menggunakan \textit{user root}.
	\item Tanda dolar \texttt{\$} menandakan Anda menggunakan \textit{user} biasa.
\end{itemize}
Jika Anda ingin menjalankan perintah administratif dari \textit{user} biasa (\texttt{\$}), tambahkan \texttt{sudo} di awal perintah \cite{shotts2019linux}.

Tabel \ref{tab:m03-dasar-teori} merangkum hal-hal penting terkait perintah dasar CLI.
\begin{table}[H]
	\caption{Tabel Ringkasan Perintah Dasar Linux}
	\label{tab:m03-dasar-teori}
	\centering
	\begin{tabular}{|l|p{9cm}|}
		\hline
		\textbf{Perintah} & \textbf{Deskripsi Singkat} \\ \hline
		\texttt{ls} & Melihat daftar file, folder di dalam direktori. \\ \hline
		\texttt{pwd} & Memeriksa \textit{path} atau alamat direktori yang saat ini diakses. \\ \hline
		\texttt{cd} & Pindah ke direktori lain. \\ \hline
		\texttt{mkdir} & Membuat direktori baru. \\ \hline
		\texttt{rm} & Menghapus file atau direktori. \\ \hline
		\texttt{cp} & Menyalin file dari satu lokasi ke lokasi lain. \\ \hline
		\texttt{mv} & Memindahkan file dari satu lokasi ke lokasi lain atau mengubah nama file. \\ \hline
		\texttt{touch} & Membuat file kosong atau memperbarui waktu akses dan modifikasi file. \\ \hline
		\texttt{cat} & Menampilkan isi file ke terminal. \\ \hline
		\texttt{nano} & Membuka editor teks nano untuk mengedit file teks. \\ \hline
		\texttt{ps} & Menampilkan daftar proses yang sedang berjalan di sistem. \\ \hline
		\texttt{kill} & Menghentikan proses yang sedang berjalan. \\ \hline
		\texttt{chmod} & Mengubah hak akses (\textit{r, w, x}) sebuah file/direktori. \\ \hline
	\end{tabular}
\end{table}

\subsection{Perangkat Keras}

\end{itemize}

\subsection{Persiapan Lingkungan}
\label{sec:m03-persiapan}
Pastikan \textit{Virtual Machine} Ubuntu Anda sudah menyala dan Anda telah berhasil \textit{login}.
\begin{enumerate}
	\item Buka terminal dengan menekan tombol \texttt{Ctrl+Alt+T} secara bersamaan.
	\item Gunakan \texttt{pwd} untuk memeriksa path atau alamat yang saat ini diakses.
	\begin{lstlisting}[language=bash, caption={Perintah memverifikasi lokasi direktori saat ini}]
		$ pwd
	\end{lstlisting}

\subsection{Sesi 1: Manipulasi Direktori dan File}
\label{sec:m03-sesi-1}
Pada sesi ini, kita akan membuat, menyalin, dan memindahkan file menggunakan perintah dasar.

\begin{enumerate}
	\item Buatlah sebuah direktori baru bernama \texttt{data} dan masuk ke dalamnya.
	\begin{lstlisting}[language=bash, caption={Membuat dan masuk ke direktori baru}]
		$ mkdir data
		$ cd data
	\end{lstlisting}
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar2.png}
		\caption{Proses pembuatan direktori baru dan berpindah ke dalamnya.}
		\label{fig:m03-gambar2}
	\end{figure}
	
	\item Membuat file teks menggunakan editor teks \texttt{nano}.
	\begin{lstlisting}[language=bash, caption={Membuka editor nano}]
		$ nano belajarlinux.txt
	\end{lstlisting}
	Ketikkan nama dan NIM Anda di dalamnya. Klik \texttt{ctrl+o} (untuk menyimpan) atau \texttt{ctrl+x} (untuk keluar), kemudian enter.
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar3.png}
		\caption{Tampilan editor teks \texttt{nano} saat mengedit file.}
		\label{fig:m03-gambar3}
	\end{figure}
	
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar4.png}
		\caption{Menampilkan isi file teks ke terminal menggunakan perintah \texttt{cat}.}
		\label{fig:m03-gambar4}
	\end{figure}
	
	\item Menampilkan isi file ke terminal menggunakan perintah \texttt{cat}.
	\begin{lstlisting}[language=bash, caption={Menampilkan isi file}]
		$ cat belajarlinux.txt
	\end{lstlisting}
	
	\item Membuat direktori baru bernama \texttt{data2} dan menyalin file ke dalamnya.
	\begin{lstlisting}[language=bash, caption={Menyalin file menggunakan cp}]
		$ mkdir data2
		$ cp belajarlinux.txt data2/
		$ ls data2/
	\end{lstlisting}
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar5.png}
		\caption{Proses penyalinan file ke dalam direktori tujuan dan verifikasinya.}
		\label{fig:m03-gambar5}
	\end{figure}
	
	\item Memindahkan file atau mengubah nama file menggunakan perintah \texttt{mv}.
	\begin{lstlisting}[language=bash, caption={Mengubah nama file menggunakan mv}]
		$ cd data2
		$ mv belajarlinux.txt belajarlagi.txt
		$ ls
	\end{lstlisting}
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar6.png}
		\caption{Pengubahan nama file menggunakan perintah \texttt{mv}.}
		\label{fig:m03-gambar6}
	\end{figure}
\end{enumerate}

\subsection{Sesi 2: Manipulasi Hak Akses}
\label{sec:m03-sesi-2}
Di sini, kita akan mencoba mengubah "kunci gembok" dari file yang sudah kita buat sebelumnya. 

\begin{enumerate}
	\item Hapus hak akses baca dan tulis untuk pengguna asing (\textit{others}) pada file \texttt{belajarlagi.txt}.
	\begin{lstlisting}[language=bash, caption={Mengubah hak akses file}]
		$ chmod o-rw belajarlagi.txt
		
		# Verifikasi perubahan hak akses, perhatikan deretan huruf di sebelah kiri
		$ ls -l
	\end{lstlisting}
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar7.png}
		\caption{Status hak akses file sebelum atau saat dilakukan modifikasi.}

	\end{figure}
	\begin{figure}[H]
		\centering
		\includegraphics[width=0.75\textwidth]{Figure/03/gambar8.png}
		\caption{Status hak akses file setelah dilakukan modifikasi dengan \texttt{chmod}.}
		\label{fig:m03-gambar8}

	\item Coba ubah kepemilikan file tersebut menjadi milik pengguna \texttt{root}. Anda memerlukan perintah \texttt{sudo} untuk melakukannya.
	\begin{lstlisting}[language=bash, caption={Menggunakan sudo untuk mengubah owner}]
		$ sudo chown root belajarlagi.txt
		$ ls -l
	\end{lstlisting}
